Como criar branchs no seu repositório:

Branchs servem para nos ajudar a dividir o que está pronto do que está sendo desenvolvido e assim não "bagunçar" o que já está funcionando.
Exemplo: Branch Master, é a nossa principal (nela contém todo o projeto)

Quando criarmos uma branch nova, a partir da master, ela terá tudo o que a master tem + as alterações que faremos nela.

No dia a dia, geralmente, é criado uma branch "feature/numero-da-task".

git checkout -b feature/numero-da-task
